# target-cwd: одна точка spawn + fail-fast (решение)

- Дата: 2026-06-26
- Статус: реализовано, Storm 0.8.0

## Контекст

Storm читал только рабочую директорию сессии: `process.cwd()` компаньона сходился
тремя путями — `spawn` без `cwd` (движки наследуют), `repoPath` лишь текстом в
промпте, песочница Gemini от `process.cwd()`. Кросс-репо аудит невозможен, а
рассинхрон молчалив. Триггер-кейс: совет запущен из `sidekey-worktree`, вопрос —
про `~/session-recall`; полезь движки читать родными tools, читали бы не тот
репозиторий (спасло лишь то, что код был вложен в промпт вручную).

## Решение

Аргумент `--cwd <path>` у компаньона → прокинут в **одну** точку, `cwd` у `spawn`
(`run-engine.mjs`). Оттуда каскадирует на все четыре движка и на песочницу Gemini.
Путь резолвится и валидируется (существующая директория) — иначе fail-fast. Echo
`repoPath` в выходном JSON. Дефолт без `--cwd` = `process.cwd()` (поведение 0.7.0
байт-в-байт).

## Почему

- **Одна точка (spawn cwd), не проводка в каждый адаптер.** claude/codex/glm берут
  рабочую дир из cwd процесса; gemini-runner строит песочницу от `process.cwd()`.
  Все наследуют `cwd`, переданный в `spawn`. Один параметр каскадирует — минимум
  поверхности изменений, меньше шанс рассинхрона текста промпта и реальной дир.
- **Fail-fast, а не молчаливый фолбэк.** Откат на `process.cwd()` при битом пути
  воспроизвёл бы корневой баг — тихий аудит не того репозитория. Громкая ошибка
  лучше тихо неверного результата.

## Что протестировали

- TDD-набор: `cwd` доходит до `spawn` (fake-engine печатает свой `process.cwd()`);
  регресс «без `--cwd` => наследование»; парсер `--cwd` (резолв, fail-fast на
  несуществующем/файле/без значения); проводка через fan-out; companion exit 2 на
  битом пути. Полный набор — 125 тестов зелёные.
- Live (2026-06-26): кросс-репо прогон на `~/session-recall` (Python) из
  sidekey-сессии (Swift). Все 4 движка `ok`, `repoPath=/Users/maxim/session-recall`,
  каждый процитировал реальные модули пакета `session_recall` (claude — `cli.py:10`/
  `server.py:61`, glm — `pyproject.toml`, gemini-agentic — список модулей через
  sandbox). Каскад `cwd → spawn → все движки + sandbox` подтверждён живьём.

## Отвергли

- **Visibility-only (echo без targeting)** — диагностирует рассинхрон, но не лечит;
  кросс-репо аудит по-прежнему невозможен.
- **Правило «запускай Storm из целевого репо» без кода** — хрупко: cwd сессии Claude
  Code фиксирован, кросс-репо (как в триггер-кейсе) не покрывается.
- **Проводка `cwd` отдельно в каждый адаптер** — избыточно; `spawn` cwd каскадирует
  через наследование сам.
- **Silent fallback на битом `--cwd`** — воспроизводит корневой баг (тихо не тот
  репо).
