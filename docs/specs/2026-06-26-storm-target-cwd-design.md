# Storm — target-cwd для совета (дизайн)

- Дата: 2026-06-26
- Статус: дизайн одобрен, спека на ревью
- Контекст: Storm 0.7.0 (claude + codex + glm + gemini), репо ~/storm
- Происхождение: реальный кейс — совет запущен из `sidekey-worktree`, а вопрос был
  про `~/session-recall`; корень найден чтением кода 0.7.0

## Контекст / проблема

Всё «куда смотреть совету» прибито к `process.cwd()` компаньона, и сходится в одну
точку тремя путями:

- `run-engine.mjs:44` — `spawn(cmd, args, { stdio, env })` **без `cwd`**: дочерние
  процессы (claude/codex/glm/gemini-runner) молча наследуют рабочую директорию
  компаньона.
- `storm-companion.mjs:17` — `repoPath: process.cwd()` идёт **лишь текстом** в
  промпт (`Repository: <path>`).
- `openrouter-runner.mjs:66` — `const cwd = process.cwd()` — корень песочницы
  Gemini = рабочая директория того же процесса.

Источник у всех трёх один — cwd сессии, откуда запущена `/storm`. Натравить совет
на репозиторий, отличный от cwd сессии, нельзя.

Кейс (2026-06-25): сессия в `sidekey-worktree`, вопрос про `~/session-recall`.
Промпт говорил движкам `Repository: sidekey-worktree`; полезь claude/codex/glm
читать родными tools — читали бы **не тот** репозиторий. Совет вышел валидным
только потому, что нужный код был вложен в промпт вручную. Это маскирует дыру, а
не закрывает её: «полная сила» (движки сами аудируют файлы) для кросс-репо
недоступна, а рассинхрон молчалив.

## Решение

Принять целевой репозиторий аргументом и прокинуть его в **одну** точку — `cwd`
у `spawn`. Оттуда он каскадирует на все четыре движка и на песочницу Gemini.
Плюс эхо фактического пути в результат — чтобы рассинхрон был виден сразу.

Дефолт (без аргумента) = `process.cwd()` — текущее поведение байт-в-байт.

### Почему одна точка (spawn cwd)

claude/codex/glm берут рабочую директорию из cwd своего процесса; gemini-runner
строит песочницу от `process.cwd()`. Все они унаследуют `cwd`, переданный в
`spawn`. Поэтому лезть в каждый адаптер не нужно — один `cwd` в `spawn`
каскадирует и на CLI-движки (правильная рабочая дир для их родных Read/Grep), и на
HTTP-движок Gemini (корень песочницы). `repoPath` в промпте при этом выставляется
из того же значения, так что текст промпта и реальная рабочая дир согласованы.

### Почему fail-fast, а не молчаливый фолбэк

Если переданный путь не существует/не директория — **падаем с явной ошибкой**, не
откатываемся на `process.cwd()`. Молчаливый фолбэк воспроизвёл бы ровно тот баг,
который чиним: совет тихо читает не тот репозиторий. Лучше громкая ошибка, чем
тихо неверный аудит.

## Компоненты (по файлам)

### `scripts/storm-companion.mjs`
- Мини-парсер argv: выделить опциональный `--cwd <path>` из `process.argv`, оставив
  `mode` и `task` как позиционные (порядок флага и задачи не важен).
- Резолв + валидация: `path.resolve(raw)` → `statSync(...).isDirectory()`; иначе
  выход с кодом ≠0 и понятным сообщением (fail-fast).
- Передать резолвнутый `cwd` в `runAll(... , { cwd })` (он же станет `repoPath`).
- Добавить `repoPath` в выходной JSON рядом с `mode/task/results`.
- Без `--cwd` → `cwd = process.cwd()` (как сейчас).

### `scripts/lib/fan-out.mjs`
- `repoPath` для промпта берётся из `opts.cwd` (сейчас `opts.repoPath`).
- Прокинуть `cwd: opts.cwd` в каждый вызов `runner(...)`.

### `scripts/lib/run-engine.mjs`
- `runEngine`/`runInvocation` принимают `opts.cwd`.
- В `spawn(cmd, args, { stdio, env })` добавить `cwd: opts.cwd`. `undefined` ⇒ Node
  наследует рабочую дир родителя (текущее поведение, регресс сохранён).

### `scripts/lib/openrouter-runner.mjs`
- Без изменений: `process.cwd()` унаследует `cwd`, переданный компаньоном в `spawn`,
  поэтому корень песочницы автоматически становится целевым репозиторием.

### `scripts/lib/adapters.mjs`
- Без изменений.

### `commands/storm.md` + `skills/storm-runtime/SKILL.md`
- Контракт оркестратора: по умолчанию целевой репозиторий = cwd сессии; если задача
  **явно про другой локальный репозиторий**, оркестратор резолвит его абсолютный
  путь и передаёт `--cwd <abs-path>`. После прогона показывает в саммари «Council
  read from: `<repoPath>`» из выходного JSON.

### `README.md`
- Короткий раздел про `--cwd`: когда нужен (кросс-репо аудит) и что по умолчанию
  читается cwd сессии.

## Контракт результата

- Выход companion получает дополнительное поле `repoPath` (абсолютный путь, который
  реально читал совет). Поля `mode/task/results` и статусы движков
  (`ok/salvaged/stalled/auth_required/timeout/no_result/error`) — без изменений.
- Без `--cwd` поведение тождественно 0.7.0 (регресс-гард в тестах).

## Безопасность (без регрессии)

- Песочница Gemini остаётся относительной cwd: выход за корень запрещён, секреты и
  `.git` блокируются — двигается лишь корень, на легитимный репозиторий
  пользователя.
- claude/codex/glm в headless `-p`/`exec` де-факто read-only (нет
  `--dangerously-skip-permissions`); смена cwd прав не добавляет — лишь даёт
  правильный репозиторий для чтения.

## Failure modes (риски и митигации)

- **Несуществующий/не-директория путь** → fail-fast с явной ошибкой (не фолбэк).
- **Относительный путь** → `path.resolve` к абсолютному относительно cwd компаньона;
  оркестратор обязан передавать абсолютный (Node не раскрывает `~`).
- **Симлинк на директорию** → `statSync` следует по ссылке, `isDirectory()` истинно;
  приемлемо.
- **Путь с пробелами** → едет элементом argv-массива (не через shell), безопасен.
- **Orphaned grandchildren на SIGKILL** — pre-existing долг, этим дизайном не
  трогается.

## Тестирование (TDD)

- `run-engine`: `runInvocation` прокидывает `cwd` в `spawn` — fake-engine печатает
  свой `process.cwd()`, тест проверяет, что вывод = переданный путь; без `cwd` —
  печатает рабочую дир процесса (наследование).
- `storm-companion` (или выделенный парсер): `--cwd` извлекается, резолвится;
  несуществующий путь → ненулевой выход/throw; `repoPath` присутствует в выходном
  JSON.
- Регресс: без `--cwd` `repoPath === process.cwd()`, поведение движков не изменилось.
- Gemini sandbox следует cwd — integration-lite: запустить runner с заданным cwd,
  убедиться, что `read_file` относителен ему (можно через существующие
  openrouter-tools тесты, передав cwd явно).

## Не входит (бэклог)

- Action-режим / изолированные эксперименты (worktree-копии репо, патч, прогон
  тестов) — отдельная большая фаза.
- Авто-детект целевого репозитория без явного `--cwd`.
- Множественные целевые директории за один прогон.
